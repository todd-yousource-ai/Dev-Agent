# Interface Contracts - ForgeAgent

This document defines the wire-format and API contract reference for the ForgeAgent subsystem, derived from the provided TRD standards and interface requirements.

## Data Structures

All structured data in the backend MUST be implemented as dataclasses.

### `ConsensusRunRequest`

Request object passed to the consensus engine.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `task` | `str` | Yes | Human-readable task description. Example pattern: `"Implement: {spec.title}"`. |
| `context` | `str` | Yes | Context string supplied to the model. External document context MUST be placed in the USER prompt, never the SYSTEM prompt. All loaded document chunks MUST pass injection scanning before inclusion. |
| `language` | `"python" \| "swift" \| "go" \| "typescript" \| "rust"` | Yes | MUST always be passed. Exact allowed values only. |

### `ConsensusRunResult`

Result returned by the consensus engine.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `final_code` | `str` | Yes | Generated code output. |
| `security_refusal` | `bool` | No | If present and `true`, processing MUST stop, gate, and log. Never retry to bypass. |

### `VTZEnforcementDecision`

Record produced when VTZ policy is evaluated.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `verdict` | `str` | Yes | For denial, MUST equal `block`. |
| `session_id` | `str` | No | Session identifier if available. |
| `ctx_id` | `str` | No | CTX-ID associated with the decision. |
| `reason` | `str` | No | Policy reason for allow/restrict/block. |
| `policy_version` | `str` | No | Policy version used for evaluation. |
| `ts` | `int` | No | UTC Unix timestamp with millisecond precision if included. |

### `TrustFlowEvent`

Synchronous audit event emitted for every action outcome.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `event_id` | `str` | Yes | MUST be globally unique. MUST be generated by CSPRNG. MUST NOT be sequential. |
| `session_id` | `str` | Yes | Session identifier. |
| `ctx_id` | `str` | Yes | Validated CTX-ID token identifier. |
| `ts` | `int` | Yes | MUST be UTC Unix timestamp with millisecond precision. |
| `event_type` | `str` | Yes | Event classification for action outcome. |
| `payload_hash` | `str` | Yes | MUST be SHA-256 of the serialized action payload. Lower-level encoding of serialized payload is implementation-defined unless otherwise specified by owner TRD. |

### `CTXIDToken`

Logical CTX-ID token contract.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `ctx_id` | `str` | Yes | Token identifier. Immutable once issued. |
| `session_id` | `str` | Yes | Session bound to the token. |
| `vtz_id` | `str` | Yes | Every agent session is bound to EXACTLY ONE VTZ at issuance. |
| `issued_at` | `int` | Yes | Token issuance time. Unix timestamp semantics are deployment-defined unless owner TRD specifies otherwise. |
| `expires_at` | `int` | Yes | Expired CTX-ID MUST be rejected. Clock skew tolerance is deployment-defined. |
| `public_key_id` | `str` | No | If present, identifies TrustLock public key used for validation. |

### `AgentActionPayload`

Serialized action payload subject to enforcement and hashing.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `session_id` | `str` | Yes | Session performing the action. |
| `ctx_id` | `str` | Yes | MUST be validated first at every entry point processing an agent action. Missing CTX-ID MUST be treated as `UNTRUSTED`. |
| `action_type` | `str` | Yes | Action identifier. |
| `payload` | `object` | Yes | Arbitrary action-specific payload. Serialized form is hashed into `payload_hash`. |
| `vtz_id` | `str` | No | If present, MUST match session-bound VTZ unless explicit cross-VTZ authorization exists. |

### `FixStrategySelectionInput`

Logical input to failure-type-aware fix loop selection.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `failure_type` | `str` | Yes | Failure category used by `_choose_strategy(failure_type, attempt, records)`. |
| `attempt` | `int` | Yes | Current retry attempt. |
| `records` | `list[object]` | Yes | Historical records consulted by strategy selection. |

### `FixArbitrationInput`

Logical input to fix arbitration scoring.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `candidate_fix` | `str` | Yes | Candidate response/code fix. |
| `assertion_tokens` | `list[str]` | Yes | Tokens used by `_score_fix()` for overlap-based scoring. |

## Validation Rules

### 1. Entry Point Enforcement

Every entry point that processes an agent action MUST:

1. Call CTX-ID validation FIRST.
2. Reject immediately on CTX-ID validation failure.
3. Perform VTZ policy check BEFORE execution.
4. Emit a TrustFlow event for every outcome: allow, restrict, block.
5. Treat TrustFlow emission failure as non-silent: log and surface the failure.

Partial processing before CTX-ID validation is non-conformant.

### 2. CTX-ID Validation

- CTX-ID tokens are IMMUTABLE once issued.
- Rotation creates a new token.
- The old token is invalidated immediately.
- Expired CTX-ID MUST be rejected.
- Validation MUST be against TrustLock public key.
- Software-only validation is rejected.
- Missing CTX-ID MUST be treated as `UNTRUSTED`.
- Identity MUST NEVER be inferred from surrounding context when CTX-ID is missing.

### 3. VTZ Enforcement

- Every agent session is bound to EXACTLY ONE VTZ at CTX-ID issuance.
- Cross-VTZ tool calls require explicit policy authorization.
- Implicit cross-VTZ access is denied.
- VTZ boundaries are structural and cannot be bypassed by application code.
- VTZ policy changes take effect at NEXT CTX-ID issuance, not mid-session.

### 4. TrustFlow Emission

Every TrustFlow event MUST include all of:

- `event_id`
- `session_id`
- `ctx_id`
- `ts`
- `event_type`
- `payload_hash`

Additional requirements:

- `event_id` MUST be globally unique.
- `event_id` MUST use CSPRNG.
- `event_id` MUST NOT be sequential.
- `ts` MUST be UTC Unix timestamp with millisecond precision.
- `payload_hash` MUST be SHA-256 of the serialized action payload.
- Emission MUST be synchronous in the enforcement path.
- Async buffering is not permitted.
- Failed emission is a `WARN`-level audit event.
- Failed emission MUST NOT be silently skipped.

### 5. Consensus Engine Invocation

ForgeAgent consensus calls MUST pass `language`:

```python
result = await self._consensus.run(
    task=f"Implement: {spec.title}",
    context=context_string,
    language=spec.language,
)
code = result.final_code
```

Language-specific system selection rules:

- `language="swift"` selects `SWIFT_GENERATION_SYSTEM` plus optional `SWIFT_UI_ADDENDUM` when UI keywords are detected.
- `language="python"` selects `GENERATION_SYSTEM`.

### 6. Fix Loop / Arbitration

- Fix loop strategy MUST be failure-type-aware via `_choose_strategy(failure_type, attempt, records)`.
- Static lookup-table strategy selection is non-conformant.
- Fix arbitration MUST use `_score_fix()` based on assertion token overlap.
- Length-based arbitration is forbidden.

Forbidden pattern:

```python
winner = max(claude_fix, openai_fix, key=len)
```

### 7. Security Rules

- Never hardcode credentials, API keys, tokens, or secrets as string literals.
- Never use `shell=True` in subprocess calls.
- Never call `eval()` or `exec()` on generated or external content.
- Never log HTTP response bodies.
- Log status codes and error types only.
- All file paths written to disk MUST pass `path_security.validate_write_path()` before any write.
- All loaded document chunks MUST pass injection scanning before inclusion in any LLM prompt.
- Context from external documents goes in the USER prompt, never the SYSTEM prompt.
- On `SECURITY_REFUSAL`, stop, gate, log, and never retry to bypass.

### 8. Backend Python Requirements

- Python version: `3.12`
- All functions MUST have type annotations.
- Backend MUST use `async/await` throughout.
- No blocking calls on the event loop.
- Structured data MUST use dataclasses.
- Tests MUST use `pytest`.
- Tests MUST live in `tests/` and mirror `src/` structure.
- `ruff` MUST pass clean.
- `mypy` MUST pass clean.
- Test coverage on all new modules MUST be `>= 85%`.
- `pyyaml` MUST be present in `requirements.txt`.

### 9. Swift-Specific Prohibitions

Forbidden patterns:

```swift
let value = optional!
```

```swift
let client = AnthropicClient(apiKey: keychainValue)
```

```swift
let token = KeychainKit.read("github_token")
```

Constraint notes:

- Only Swift reads Keychain.
- Swift reads Keychain only to deliver via XPC.
- Backend Python reads nothing from Keychain.
- LLM API calls from Swift are forbidden.

## Invalid payloads

### Invalid `ConsensusRunRequest` — missing `language`

```json
{
  "task": "Implement: Add audit logging",
  "context": "Scanned context"
}
```

Reason: `language` MUST always be passed.

### Invalid `ConsensusRunRequest` — unsupported `language`

```json
{
  "task": "Implement: Add audit logging",
  "context": "Scanned context",
  "language": "java"
}
```

Reason: allowed values are only `"python"`, `"swift"`, `"go"`, `"typescript"`, `"rust"`.

### Invalid `TrustFlowEvent` — missing required fields

```json
{
  "event_id": "123",
  "session_id": "sess_123",
  "event_type": "allow"
}
```

Reasons:

- missing `ctx_id`
- missing `ts`
- missing `payload_hash`

### Invalid `VTZEnforcementDecision` — denial without `block`

```json
{
  "verdict": "deny",
  "session_id": "sess_123",
  "ctx_id": "ctx_456"
}
```

Reason: VTZ policy denial MUST produce a `VTZEnforcementDecision` with `verdict=block`.

### Invalid agent action processing flow

```json
{
  "session_id": "sess_123",
  "action_type": "tool_call",
  "payload": {
    "tool": "repo.write_file"
  }
}
```

Reason: missing `ctx_id` MUST be treated as `UNTRUSTED`; identity must not be inferred from context.

## Integration Points

### Consensus Engine

Invocation contract:

- Method shape: `await self._consensus.run(task=..., context=..., language=...)`
- `language` is mandatory.
- Consumer reads generated code from `result.final_code`.

### Enforcement Pipeline

Required processing order for agent actions:

1. CTX-ID validation
2. Immediate reject on validation failure
3. VTZ policy evaluation
4. Execution only if allowed/restricted per policy
5. Synchronous TrustFlow emission for outcome
6. Log and surface TrustFlow emission failure

### Path Security

Before any disk write:

- Call `path_security.validate_write_path()` on every file path.

### Prompt Construction

- External document chunks MUST be injection scanned before prompt inclusion.
- External document context MUST be placed in the USER prompt.
- External document context MUST NOT be placed in the SYSTEM prompt.

### Security Gate Handling

If LLM output indicates `SECURITY_REFUSAL`:

1. Stop
2. Gate
3. Log
4. Do not retry to bypass

### Tooling and Test Integration

Required project-level integration constraints:

- Test command before changes:

```bash
cd src && pytest ../tests/ -v --tb=short
```

- Testing framework: `pytest`
- Linting: `ruff`
- Type checking: `mypy`

### Conformance Notes

A ForgeAgent component is non-conformant if it violates any of the following hard requirements:

- missing CTX-ID-first validation
- partial processing before CTX-ID validation
- VTZ check after execution
- omission of `verdict=block` on VTZ denial
- omission of TrustFlow emission for any action outcome
- silent TrustFlow emission failure
- omitted or invalid `language` in consensus calls
- length-based fix arbitration
- software-only CTX-ID validation
- implicit identity inference when CTX-ID is missing
- async buffering of TrustFlow emission in the enforcement path
