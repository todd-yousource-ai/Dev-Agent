

# Code Conventions — Forge Platform

> **Scope:** All code authored by human engineers and generated by the ConsensusDevAgent subsystem.
> Every rule is enforced in CI. No exceptions without a signed-off waiver in `docs/waivers/`.

---

## 1. File and Directory Naming

1. **Source directories** use short, lowercase slugs with no separators:
   ```
   src/cal/           # Conversation Abstraction Layer
   src/dtl/           # Data Trust Label
   src/trustflow/     # TrustFlow audit stream
   src/vtz/           # Virtual Trust Zone enforcement
   src/trustlock/     # Cryptographic machine identity (TPM-anchored)
   src/mcp/           # MCP Policy Engine
   src/rewind/        # Forge Rewind replay engine
   sdk/connector/     # Forge Connector SDK
   ```

2. **Test directories** mirror `src/` exactly:
   ```
   tests/cal/
   tests/dtl/
   tests/trustflow/
   tests/vtz/
   tests/trustlock/
   tests/mcp/
   tests/rewind/
   tests/connector/
   ```

3. **Python files** use `snake_case.py`. Never use hyphens.
   ```
   src/dtl/label_verifier.py       ✅
   src/dtl/label-verifier.py       ❌
   ```

4. **TypeScript/Swift files** use `PascalCase` for components/classes, `camelCase` for utilities:
   ```
   StreamGateCard.swift             ✅  (SwiftUI view)
   parseGatePayload.ts              ✅  (utility)
   ```

5. **Config, schema, and manifest files** use `snake_case` with their natural extension:
   ```
   consensus_config.yaml
   build_manifest.json
   trust_label_schema.json
   ```

6. **One module per concern.** A file that exceeds 400 lines must be split, with each new file covering a single responsibility.

## 2. Branch Naming

7. **All agent-authored branches** follow this exact pattern — no deviations:
   ```
   forge-agent/build/{engineer_id}/{subsystem_slug}/pr-{N:03d}-{title_slug}
   ```
   - `engineer_id`: the operator's Forge engineer ID (lowercase, alphanumeric + hyphens).
   - `subsystem_slug`: one of `cal`, `dtl`, `trustflow`, `vtz`, `trustlock`, `mcp`, `rewind`, `connector`.
   - `N`: zero-padded three-digit PR sequence number.
   - `title_slug`: kebab-case summary, max 48 characters.

   ```
   forge-agent/build/eng-0042/dtl/pr-007-add-label-expiry-check   ✅
   feature/add-label-expiry                                         ❌
   ```

8. **Human feature branches** use `forge/{engineer_id}/{subsystem_slug}/{description}`. The `forge-agent/` prefix is reserved exclusively for ConsensusDevAgent output.

## 3. Class and Function Naming

9. **Python classes**: `PascalCase`. Suffix with role when ambiguous:
   ```python
   class LabelVerifier:          # core domain
   class LabelVerifierError:     # exception (see §5)
   class LabelVerifierFixture:   # test fixture factory
   ```

10. **Python functions and methods**: `snake_case`, verb-first:
    ```python
    def verify_label(label: TrustLabel) -> VerificationResult: ...
    def build_consensus_plan(intent: str) -> ConsensusPlan: ...
    ```

11. **Private members**: single leading underscore. Never use double underscores for name-mangling unless wrapping a C extension.
    ```python
    def _resolve_scope(self) -> str: ...
    ```

12. **Constants**: `UPPER_SNAKE_CASE`, declared at module level:
    ```python
    MAX_CONSENSUS_ROUNDS = 5
    DEFAULT_GATE_TIMEOUT_S = 30
    ```

13. **TypeScript** follows the same spirit: `PascalCase` classes/interfaces/types, `camelCase` functions/variables, `UPPER_SNAKE_CASE` constants.

14. **Swift accessibility identifiers** follow the `axIdentifier` convention — set via `.accessibilityIdentifier()` on every interactive element:
    ```
    {module}-{component}-{role}-{context?}
    ```
    Examples:
    ```
    "auth-touchid-button"
    "auth-passcode-button"
    "settings-anthropic-key-field"
    "settings-anthropic-key-test-button"
    "settings-anthropic-key-reveal-button"
    "navigator-project-row-{projectId}"
    "stream-gate-card-{gateId}"
    "stream-gate-yes-button-{gateId}"
    "stream-gate-skip-button-{gateId}"
    "stream-gate-stop-button-{gateId}"
    ```
    Omit `{context}` only when the component is globally unique.

## 4. Import and Module Organisation

15. **Import order** (enforced by `isort` profile `black`):
    1. Standard library
    2. Third-party packages
    3. Forge platform (`src/...`)
    4. Local relative imports

    Separate each group with a single blank line.

16. **No wildcard imports.** `from module import *` is banned in all production and test code.

17. **Absolute imports from `src/`** in production code:
    ```python
    from src.dtl.label_verifier import LabelVerifier      ✅
    from ..label_verifier import LabelVerifier             ❌  (in prod)
    ```
    Relative imports are permitted only within a single subsystem's test package.

18. **Lazy imports** for heavy optional dependencies (ML models, large SDKs). Guard with `TYPE_CHECKING`:
    ```python
    from __future__ import annotations
    from typing import TYPE_CHECKING

    if TYPE_CHECKING:
        from heavy_sdk import HeavyClient
    ```

## 5. Error and Exception Patterns

19. **Every subsystem defines a base exception** in `<subsystem>/exceptions.py`:
    ```python
    class DtlError(Exception):
        """Base for all DTL subsystem errors."""
    ```

20. **Derived exceptions** are specific and named `{Noun}{Verb}Error`:
    ```python
    class LabelVerificationError(DtlError): ...
    class LabelExpiredError(DtlError): ...
    class ConsensusQuorumNotReachedError(ConsensusError): ...
    ```

21. **Never catch bare `Exception` or `BaseException`** unless re-raising or at an explicit top-level boundary (CLI entry point, HTTP handler).

22. **Always chain exceptions** when wrapping:
    ```python
    except KeyError as exc:
        raise LabelFieldMissingError(field_name) from exc
    ```

23. **Path validation before every write.** This is a security-critical, non-negotiable rule:
    ```python
    from path_security import validate_write_path

    safe_path = validate_write_path(user_supplied_path)  # raises on traversal
    with open(safe_path, "w") as f:
        f.write(content)
    ```
    Any write operation that skips `validate_write_path` will fail the `forge-security` CI gate.

24. **Agent-generated code must wrap all subprocess and file I/O** in a try/except that catches `OSError` and `subprocess.CalledProcessError`, logs the failure, and raises a subsystem-specific exception.
