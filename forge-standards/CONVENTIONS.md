# CONVENTIONS.md — ForgeAgent Subsystem

All rules below are derived from the ForgeAgent TRD documents. Every rule is mandatory unless explicitly marked otherwise.

---

## 1. File and Directory Naming

1. **Source directories use short, lowercase subsystem slugs.** Each top-level package maps to exactly one subsystem:

   | Directory | Subsystem |
   |---|---|
   | `src/cal/` | Conversation Abstraction Layer |
   | `src/dtl/` | Data Trust Label |
   | `src/trustflow/` | TrustFlow audit stream |
   | `src/vtz/` | Virtual Trust Zone enforcement |
   | `src/trustlock/` | Cryptographic machine identity (TPM-anchored) |
   | `src/mcp/` | MCP Policy Engine |
   | `src/rewind/` | Forge Rewind replay engine |
   | `sdk/connector/` | Forge Connector SDK |

2. **Tests mirror `src/` exactly.** A module at `src/vtz/policy.py` has its tests at `tests/vtz/test_policy.py`. No exceptions.

3. **Python backend files use `snake_case.py`.** Named modules at the `src/` root follow the established pattern:
   - `src/consensus.py` — `ConsensusEngine`, `GENERATION_SYSTEM`, `SWIFT_GENERATION_SYSTEM`, `UI_ADDENDUM`
   - `src/build_director.py` — `BuildPipeline`

4. **Repo-root files are validated by basename, not by directory.** The following files must live at the repository root and are checked by name only:
   `README.md`, `CODEOWNERS`, `LICENSE`, `.gitignore`, `.editorconfig`, `VERSION`, `Makefile`, `pyproject.toml`.

5. **CI workflow files use the canonical names:**
   - `crafted-ci.yml` — Ubuntu pipeline.
   - `crafted-ci-macos.yml` — macOS Swift pipeline.

6. **`conftest.py` is auto-committed.** The file is generated by `ci_workflow.ensure()` to enable `src/` imports. Never hand-edit the auto-generated `conftest.py`.

---

## 2. Branch Naming

7. **All ForgeAgent branches follow this exact pattern:**

   ```
   forge-agent/build/{engineer_id}/{subsystem_slug}/pr-{N:03d}-{title_slug}
   ```

   - `engineer_id` — lowercase alphanumeric, no dots.
   - `subsystem_slug` — matches a directory name from Rule 1 (e.g., `trustflow`, `vtz`).
   - `N` — zero-padded to 3 digits (e.g., `001`, `042`).
   - `title_slug` — lowercase kebab-case summary, max 48 characters.

   The prefix `forge-agent` (not `forge_agent`) is kept intentionally for compatibility. Do not rename it.

---

## 3. Class and Function Naming

8. **Classes use `PascalCase`.** Suffix with the subsystem role when ambiguous: `ConsensusEngine`, `BuildPipeline`, `TrustFlowAuditor`.

9. **Functions and methods use `snake_case`.** Public API functions are verb-first: `validate_write_path()`, `ensure_conftest()`, `resolve_gate()`.

10. **Constants use `UPPER_SNAKE_CASE`.** Module-level constants are defined at the top of the file, below imports: `GENERATION_SYSTEM`, `SWIFT_GENERATION_SYSTEM`, `UI_ADDENDUM`.

11. **Private/internal names are single-underscore prefixed.** Example: `_is_docs_pr`, `_docs_keywords`. Never use double-underscore name-mangling unless required by inheritance conflict.

---

## 4. SwiftUI Accessibility Identifier Naming

12. **All interactive SwiftUI elements must have an `.accessibilityIdentifier()`.** No interactive control ships without one.

13. **Identifiers follow the pattern `{module}-{component}-{role}-{context?}`:**

    ```
    auth-touchid-button
    auth-passcode-button
    settings-anthropic-key-field
    settings-anthropic-key-test-button
    settings-anthropic-key-reveal-button
    navigator-project-row-{projectId}
    stream-gate-card-{gateId}
    stream-gate-yes-button-{gateId}
    stream-gate-skip-button-{gateId}
    stream-gate-stop-button-{gateId}
    ```

14. **All segments are lowercase kebab-case.** Dynamic suffixes (e.g., `{projectId}`, `{gateId}`) are appended after a hyphen and use the runtime value directly.

## 5. Error and Exception Patterns

15. **Validate every path before any write operation.** Use the `path_security` module; never write to a user-supplied path without validation:

    ```python
    from path_security import validate_write_path

    safe_path = validate_write_path(user_supplied_path)
    # Returns a safe default on directory-traversal attempt.
    ```

16. **Never silently swallow exceptions.** Every `except` block must either re-raise, log at `WARNING` or above, or return a well-defined error object. Bare `except:` and `except Exception: pass` are forbidden.

17. **Custom exceptions inherit from a single subsystem base class.** Each subsystem defines one base (e.g., `class VTZError(Exception)`) and all specific errors inherit from it.

---

## 6. Import and Module Organisation

18. **Import order (enforced by linter):**
    1. Standard library
    2. Third-party packages
    3. `src/` internal modules (absolute imports from `src`)
    4. Local relative imports (only within the same subsystem package)

    Separate each group with a single blank line.

19. **Absolute imports from `src/` are preferred.** Use `from src.vtz.policy import enforce` rather than deep relative imports across subsystem boundaries.

20. **Never import from `conftest.py`.** It exists solely for pytest path setup.

21. **Keyword-list constants for classification logic are defined inline as frozen sets, not module-level lists:**

    ```python
    _docs_keywords: frozenset[str] = frozenset({
        "naming convention", "glossary", "changelog",
    })
    ```

    (The v5.0 keyword-list approach was removed in v6.0; if keyword matching is reintroduced, use `frozenset` and keep it private.)

---

## 7. Comment and Documentation Rules

22. **Every public function and class has a docstring.** Use imperative mood for the first line: `"""Validate the supplied path against traversal attacks."""`

23. **Inline comments explain *why*, not *what*.** The code shows what; the comment justifies non-obvious decisions.

24. **Convention references in comments use the tag format `# Convention: ...`:**

    ```swift
    // Convention: {module}-{component}-{role}-{context?}
    ```

25. **TRD traceability.** When a block of code implements a specific TRD requirement, add a comment in the form `# TRD-{N}: {section title}` on the line above.

---

## 8. ForgeAgent-Specific Patterns

26. **Path security is a hard gate.** Any ForgeAgent operation that writes to disk must call `validate_write_path()` as the first statement before constructing file handles or invoking OS write calls. This is non-negotiable.

27. **PR classification logic must not rely on mutable keyword lists.** Title/body matching uses deterministic, version-pinned logic. If keyword sets are used, they are `frozenset` constants reviewed in the same PR that changes classification behaviour.

28. **`ci_workflow.ensure()` is the single entry point for CI file generation.** Do not manually create or modify `conftest.py`, `crafted-ci.yml`, or `crafted-ci-macos.yml`. All three are outputs of the pipeline ensure step.
