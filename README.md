# {repo}

This repository contains the implementation and source specifications for an enterprise security platform that enforces architecture-defined controls, trust boundaries, and interface contracts across its core security subsystems.

## What It Does
The platform implements the security architecture defined in the TRDs and standards in this repository, turning documented controls into enforceable behavior in `src/` and verifiable behavior in `tests/`. It is designed for developers and reviewers working on high-assurance services where subsystem boundaries, protocol behavior, and policy enforcement must remain consistent with the architecture.

## Key Subsystems
- Architecture Standards — defines system-wide rules, constraints, and decision boundaries
- Interface Contracts — defines wire formats, API contracts, and subsystem interaction rules
- Implementation Layer — applies the documented controls in runnable code under `src/`
- Test Suite — verifies enforcement behavior, integration expectations, and regressions
- CI Workflows — runs validation and quality gates in `.github/workflows/`
- LLM Development Guidance — constrains AI-assisted code generation via `CLAUDE.md`

## Architecture Overview
The repository is organized around specification-first enforcement: product and technical requirements in `forge-docs/` define expected behavior, and architecture standards in `forge-standards/` define the rules implementation must follow. Development flows from standards to interfaces to code to tests, with CI acting as the final enforcement layer to keep implementations aligned with the documented trust model and contracts.

## Repository Structure
```text
forge-docs/          — source TRDs and PRDs
forge-standards/     — architecture, interfaces, decisions, conventions
CLAUDE.md            — LLM coding instructions (read this first)
src/                 — implementation (generated by Crafted Dev Agent)
tests/               — test suite
.github/workflows/   — CI
```

## Getting Started
- Read `CLAUDE.md` first; it defines the required development and code-generation constraints for this repository.
- Review `forge-standards/ARCHITECTURE.md` to understand the enforcement model, subsystem boundaries, and architectural rules.
- Review `forge-standards/INTERFACES.md` before changing any integration point, message shape, or API behavior.
- Use `forge-docs/` as the source of truth for product and technical requirements before implementing features in `src/`.
- Validate changes against `tests/` and CI workflows to ensure the implementation still conforms to the documented standards.
