## What

Establishes the foundational product identity specification for the Consensus Dev Agent project. This PR introduces three artifacts:

1. **`docs/product_identity/naming_policy.md`** — Human-readable naming policy document defining the product name hierarchy, approved derived forms, context-specific usage rules, and explicitly prohibited legacy variants.
2. **`shared/product_identity/canonical_name_spec.json`** — Machine-readable canonical name specification that serves as the single source of truth from which both Python and Swift constants modules will be derived in subsequent PRs.
