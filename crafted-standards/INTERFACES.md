# Interface Contracts - CraftedApp

Definitive wire-format and API contract reference for the CraftedApp subsystem.

## Data Structures

### TrustFlowEvent

Required for every action outcome emitted in the enforcement path.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `event_id` | `string` | Yes | MUST be globally unique. MUST be generated by CSPRNG. MUST NOT be sequential. |
| `session_id` | `string` | Yes | Session identifier for the agent session. |
| `ctx_id` | `string` | Yes | CTX-ID token identifier associated with the session/action. |
| `ts` | `number` | Yes | UTC Unix timestamp with millisecond precision. |
| `event_type` | `string` | Yes | Event type name. No closed enum specified in source. |
| `payload_hash` | `string` | Yes | MUST be SHA-256 of the serialized action payload. |

#### Contract
- Every TrustFlow event MUST include exactly the required fields above.
- Emission MUST be synchronous in the enforcement path.
- Async buffering is not permitted.
- Failed emission is a WARN-level audit event, not a silent skip.
- TrustFlow emission failure MUST NOT silently continue — log and surface the failure.

---

### VTZEnforcementDecision

Produced when VTZ policy denies an action.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `verdict` | `string` | Yes | MUST equal `block` for VTZ policy denial. |

#### Contract
- Every action MUST be checked against VTZ policy BEFORE execution.
- VTZ policy denial MUST produce a `VTZEnforcementDecision` record with `verdict=block`.

---

### CTX-ID Token

Opaque token representing validated identity and VTZ binding.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `ctx_id` | `string` | Yes | Immutable once issued. Rotation creates a new token; old token is invalidated immediately. |

#### Contract
- CTX-ID tokens are IMMUTABLE once issued.
- Expired CTX-ID MUST be rejected.
- Clock skew tolerance is defined per deployment.
- CTX-ID MUST be validated against TrustLock public key.
- Software-only validation is rejected.
- Missing CTX-ID MUST be treated as `UNTRUSTED`.
- Never infer identity from context when CTX-ID is missing.

---

### Agent Action Payload

Serialized payload processed by an entry point and hashed into `payload_hash`.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| _implementation-defined_ | _object_ | Yes | Serialized form MUST be stable enough to compute SHA-256 for `payload_hash`. |

#### Contract
- Every entry point that processes an agent action MUST call CTX-ID validation FIRST.
- CTX-ID validation failure MUST result in immediate rejection.
- No partial processing is permitted.
- Every action outcome (`allow`, `restrict`, `block`) MUST emit a TrustFlow event.

---

### ConsensusRunRequest

Invocation contract for the consensus engine.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `task` | `string` | Yes | Example source usage: `f"Implement: {spec.title}"`. |
| `context` | `string` | Yes | Context string passed to the engine. |
| `language` | `string` | Yes | MUST be one of: `"python"`, `"swift"`, `"go"`, `"typescript"`, `"rust"`. |

#### Source-defined usage
```python
result = await self._consensus.run(
    task=f"Implement: {spec.title}",
    context=context_string,
    language=spec.language,   # "python" | "swift" | "go" | "typescript" | "rust"
)
code = result.final_code
```

---

### ConsensusRunResult

Result returned by the consensus engine.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `final_code` | `string` | Yes | Final generated code output. |

---

### Fix Strategy Selection Input

Internal decision input for failure-aware fix strategy selection.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `failure_type` | `string` | Yes | Failure classifier input to `_choose_strategy(failure_type, attempt, records)`. |
| `attempt` | `integer` | Yes | Retry/fix attempt count. |
| `records` | `array` | Yes | Prior failure/fix records. Element structure not specified in source. |

#### Contract
- Fix loop strategy is failure-type-aware via `_choose_strategy(failure_type, attempt, records)`.
- Static lookup table strategy selection is non-conformant.

---

### Fix Arbitration Inputs

Internal arbitration inputs for selecting a fix candidate.

| Field | Type | Required | Constraints |
|---|---|---:|---|
| `assertion_token_overlap` | `number` | Yes | Scoring basis used by `_score_fix()`. |
| `candidate_fix` | `string` | Yes | Candidate fix content. |

#### Contract
- Fix arbitration MUST use `_score_fix()`.
- Length-based arbitration is forbidden.

## Validation Rules

### CAL Enforcement Contract
1. Every entry point that processes an agent action MUST call CTX-ID validation FIRST.
2. CTX-ID validation failure MUST result in immediate rejection.
3. No partial processing is permitted after CTX-ID validation failure.
4. Every action MUST be checked against VTZ policy BEFORE execution.
5. VTZ policy denial MUST produce a `VTZEnforcementDecision` record with `verdict=block`.
6. Every action outcome (`allow`, `restrict`, `block`) MUST emit a TrustFlow event.
7. TrustFlow emission failure MUST NOT silently continue.
8. TrustFlow emission failure MUST be logged and surfaced.

### TrustFlow Emission Contract
1. Every TrustFlow event MUST include:
   - `event_id`
   - `session_id`
   - `ctx_id`
   - `ts`
   - `event_type`
   - `payload_hash`
2. `event_id` MUST be globally unique.
3. `event_id` MUST be CSPRNG-generated.
4. `event_id` MUST NOT be sequential.
5. `ts` MUST be a UTC Unix timestamp with millisecond precision.
6. `payload_hash` MUST be the `SHA-256` of the serialized action payload.
7. Emission MUST be synchronous in the enforcement path.
8. Async buffering is not permitted.
9. Failed emission is a `WARN`-level audit event.
10. Failed emission MUST NOT be silently skipped.

### CTX-ID Contract
1. CTX-ID tokens are IMMUTABLE once issued.
2. CTX-ID rotation creates a new token.
3. The old token is invalidated immediately upon rotation.
4. Expired CTX-ID MUST be rejected.
5. Clock skew tolerance is deployment-defined.
6. CTX-ID MUST be validated against TrustLock public key.
7. Software-only validation is rejected.
8. Missing CTX-ID MUST be treated as `UNTRUSTED`.
9. Identity MUST NEVER be inferred from surrounding context when CTX-ID is missing.

### VTZ Enforcement Contract
1. Every agent session is bound to EXACTLY ONE VTZ at CTX-ID issuance.
2. Cross-VTZ tool calls require explicit policy authorization.
3. Implicit cross-VTZ authorization is denied.
4. VTZ boundaries are structural, not advisory.
5. Enforcement cannot be bypassed by application code.
6. VTZ policy changes take effect at NEXT CTX-ID issuance.
7. VTZ policy changes do not take effect mid-session.

### Consensus Engine Validation
1. `language` is required on every consensus engine invocation.
2. `language` MUST be one of:
   - `"python"`
   - `"swift"`
   - `"go"`
   - `"typescript"`
   - `"rust"`
3. `language="swift"` selects `SWIFT_GENERATION_SYSTEM` plus optional `SWIFT_UI_ADDENDUM` when UI keywords are detected.
4. `language="python"` selects `GENERATION_SYSTEM`.

### Forbidden Implementations
The following are non-conformant:

#### Length-based fix arbitration
```python
winner = max(claude_fix, openai_fix, key=len)  # WRONG — use _score_fix()
```

#### Swift force unwrap
```swift
let value = optional!
```

#### LLM API call from Swift
```swift
let client = AnthropicClient(apiKey: keychainValue)
```

#### Keychain read for backend
```swift
let token = KeychainKit.read("github_token")  // Python reads nothing from Keychain
```

### Security Rules
1. Never hardcode credentials, API keys, tokens, or secrets as string literals.
2. Never use `shell=True` in subprocess calls.
3. Never call `eval()` or `exec()` on any generated or external content.
4. Never log HTTP response bodies.
5. Log status codes and error types only for HTTP failures.
6. All file paths written to disk MUST pass `path_security.validate_write_path()` before any write.
7. All loaded document chunks MUST pass injection scanning before inclusion in any LLM prompt.
8. Context from external documents goes in the USER prompt, never the SYSTEM prompt.
9. When `SECURITY_REFUSAL` output is returned from the LLM: stop, gate, log. Never retry to bypass.

### Python Backend Requirements
1. Python version: `3.12`.
2. Type annotations are required on every function.
3. `async/await` throughout the backend.
4. No blocking calls on the event loop.
5. Dataclasses for all structured data.
6. Tests MUST use `pytest`.
7. Tests live in `tests/` and mirror `src/` structure.
8. `ruff` must pass clean.
9. `mypy` must pass clean.
10. Test coverage MUST be `>= 85%` on all new modules.
11. `pyyaml` must be present in `requirements.txt`.

## Integration Points

### Enforcement Path
Sequence requirements for any entry point processing an agent action:

1. Validate CTX-ID first.
2. If CTX-ID validation fails: immediately reject.
3. Do not partially process the action.
4. Evaluate VTZ policy before execution.
5. If denied, emit `VTZEnforcementDecision` with `verdict=block`.
6. Emit TrustFlow event for the outcome.
7. TrustFlow emission is synchronous.
8. If emission fails, log and surface the failure at `WARN` audit level.

### TrustLock
- CTX-ID validation MUST be performed against TrustLock public key.
- Software-only validation is rejected.

### VTZ Policy Engine
- Every agent session is bound to exactly one VTZ at CTX-ID issuance.
- Cross-VTZ tool calls require explicit policy authorization.
- Application code cannot bypass VTZ boundaries.

### Consensus Engine
Invocation contract:
- Must call `self._consensus.run(...)`.
- Must pass `task`.
- Must pass `context`.
- Must pass `language`.

Selection behavior:
- `language="swift"` selects `SWIFT_GENERATION_SYSTEM`.
- `SWIFT_UI_ADDENDUM` is optionally injected when UI keywords are detected.
- `language="python"` selects `GENERATION_SYSTEM`.

Returned result:
- Read generated code from `result.final_code`.

### Path Security
Before any disk write:
- All file paths MUST pass `path_security.validate_write_path()`.

### Injection Scanning
Before including loaded document chunks in any LLM prompt:
- All loaded document chunks MUST pass injection scanning.

### Prompt Placement
- Context from external documents MUST go in the USER prompt.
- Context from external documents MUST NEVER go in the SYSTEM prompt.

### Security Refusal Handling
If LLM output is exactly or includes `SECURITY_REFUSAL`:
1. Stop.
2. Gate.
3. Log.
4. Never retry to bypass.
